# 線上版部署說明

建議使用 Streamlit Community Cloud，因為這個專案本來就是 Streamlit 程式，部署流程最簡單。

## 需要準備

- GitHub 帳號
- Streamlit Community Cloud 帳號
- 本專案所有檔案：
  - `app.py`
  - `risk_rules.json`
  - `requirements.txt`
  - `README.md`
  - `.streamlit/config.toml`

## 部署步驟

1. 到 GitHub 建立一個新的 Repository。
2. 將本資料夾內的檔案上傳到 Repository。
3. 到 Streamlit Community Cloud。
4. 選擇 `New app` 或建立新應用程式。
5. 選擇剛剛建立的 GitHub Repository。
6. Main file path 填：

```text
app.py
```

7. 按下 Deploy。
8. 部署完成後，Streamlit 會產生一個可分享的網址。

## 注意事項

- 不要上傳 `.venv` 資料夾。
- `risk_rules.json` 必須和 `app.py` 放在同一層。
- 若日後修改規則，只要更新 GitHub 上的 `risk_rules.json`，Streamlit Cloud 會重新部署。
- 本程式未使用外部資料庫，部署後產生內容只在使用者當次瀏覽器操作中使用。

## 線上版與本機版差異

- 本機版網址通常是 `http://localhost:8501`，只有你的電腦可直接使用。
- 線上版會得到公開網址，例如 `https://your-app-name.streamlit.app`，其他人可直接開啟使用。
